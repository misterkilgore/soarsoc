2. OBIETTIVO DELL'ESERCIZIO
---------------------------
In questo esercizio, affronteremo un aspetto fondamentale del SOC: la gestione e il monitoraggio dei ticket di sicurezza.

Obiettivi principali:
- Visualizzare i ticket in una tabella chiara e consultabile.
- Modificare lo stato di ogni ticket per riflettere l’avanzamento delle attività di risposta.
- Visualizzare grafici a barre per analizzare rapidamente la distribuzione dei ticket per gravità (severity) e stato (status).

Questa attività aiuta a comprendere come mantenere sotto controllo gli incidenti di sicurezza, facilitare la collaborazione tra analisti e migliorare i tempi di risposta.

3. SVILUPPO DELL'ESERCIZIO
--------------------------
Sviluppare un’applicazione Streamlit che:

1. Legga il file `ticket.csv` contenente i ticket di sicurezza.
2. Mostri una tabella con tutti i ticket.
3. Permetta di cambiare lo stato di ogni ticket tramite un menu a tendina.
4. Visualizzi due grafici a barre:
   - Numero di ticket per livello di severità (LOW, MEDIUM, HIGH).
   - Numero di ticket per stato (OPEN, IN_PROGRESS, RESOLVED, CLOSED).
5. Consenta di salvare le modifiche dello stato dei ticket nella sessione.

4. DRITTE STREAMLIT E PANDAS
-----------------------------
Per creare l’applicazione si consiglia di usare:

- **pandas** per la lettura e manipolazione del file CSV.
- **streamlit** per costruire l’interfaccia web interattiva.
- **matplotlib** per i grafici a barre.

Esempi utili:

```python
import streamlit as st
import pandas as pd
import matplotlib.pyplot as plt

# Caricamento dati
df = pd.read_csv('ticket.csv')

# Visualizzazione tabella
st.dataframe(df)

# Grafico a barre
status_counts = df['status'].value_counts()
colors = {'OPEN': 'blue', 'WIP': 'orange', 'CLOSED': 'green'}
fig, ax = plt.subplots()
bars = ax.bar(status_counts.index, status_counts.values, color=[colors.get(x, 'gray') for x in status_counts.index])
ax.set_title("Ticket per Status")
ax.set_xlabel("Status")
ax.set_ylabel("Count")
st.pyplot(fig)
```
